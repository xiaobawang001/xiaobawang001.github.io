# Docker指令速查手册

## 1. 基础容器状态操作

场景：容器创建、启停、暂停、删除等生命周期管理

|命令|说明|
|---|---|
|`docker run -d --name mynginx nginx`|创建并后台运行一个容器|
|`docker start mynginx`|启动已停止的容器|
|`docker stop mynginx`|优雅停止运行中的容器|
|`docker restart mynginx`|重启容器|
|`docker pause mynginx`|暂停容器内全部进程|
|`docker unpause mynginx`|恢复已暂停的容器|
|`docker rm mynginx`|删除已停止的容器|
|`docker rm -f mynginx`|⚠️强制删除正在运行的容器|
---
## 2. 镜像管理

场景：拉取、构建、打标、推送、删除本地镜像

|命令|说明|
|---|---|
|`docker pull nginx:alpine`|从镜像仓库拉取指定版本镜像|
|`docker images`|列出本地全部镜像|
|`docker image ls`|列出本地全部镜像（新版标准写法）|
|`docker rmi nginx:alpine`|删除指定镜像|
|`docker rmi $(docker images -q)`|⚠️删除本地所有镜像（高危慎用）|
|`docker build -t myapp:v1 .`|基于当前目录 Dockerfile 构建镜像并打标签|
|`docker tag nginx:alpine myrepo/nginx:v1`|为已有镜像新增仓库标签|
|`docker push myrepo/nginx:v1`|将镜像推送到远程私有仓库|
|`docker system prune -a`|⚠️一键清理未使用镜像、容器、网络（谨慎执行）|
---
## 3. 容器信息查看与日志排查

场景：查看运行状态、资源占用、日志、进程用于问题排查

|命令|说明|
|---|---|
|`docker ps`|仅列出正在运行的容器|
|`docker ps -a`|列出所有容器（包含已停止）|
|`docker stats`|实时监控所有容器 CPU、内存、网络资源占用|
|`docker logs mynginx`|查看容器全部历史日志|
|`docker logs -f mynginx`|实时持续跟踪容器日志输出|
|`docker logs --tail 100 mynginx`|只查看容器最后 100 行日志|
|`docker inspect mynginx`|JSON 格式输出容器完整配置详情|
|`docker top mynginx`|查看容器内部运行的进程列表|
---
## 4. 进入容器调试

场景：登录容器内部修改配置、调试程序

|命令|说明|
|---|---|
|`docker exec -it mynginx /bin/bash`|交互式进入容器 bash 终端（标准推荐）|
|`docker exec -it mynginx /bin/sh`|进入 alpine 极简镜像（无 bash）|
|`docker exec mynginx ls /app`|在容器内执行单条命令，不开启交互终端|
|`docker attach mynginx`|挂载容器标准输出流（极少使用，退出易停容器）|
---
## 5. 网络管理

场景：端口映射、自定义网桥、容器互通、网络查询

|命令|说明|
|---|---|
|`docker network ls`|列出本地所有 Docker 网络|
|`docker network create mynet`|创建自定义桥接网络|
|`docker run --network mynet nginx`|指定自定义网络启动容器|
|`docker network connect mynet mynginx`|将运行中的容器加入指定网络|
|`docker network disconnect mynet mynginx`|将容器从网络中断开|
|`docker network rm mynet`|删除闲置自定义网络|
|`docker network inspect mynet`|查看网络详细配置与接入容器|
|`docker port mynginx`|查看容器端口映射关系|
|`docker run -p 8080:80 nginx`|端口映射：宿主机 8080 转发容器 80 端口|
---
## 6. 数据持久化与文件拷贝

场景：Volume 持久化存储、宿主机目录挂载、容器内外文件互传

|命令|说明|
|---|---|
|`docker volume ls`|列出所有数据卷|
|`docker volume create mydata`|创建命名持久化数据卷|
|`docker volume inspect mydata`|查看数据卷存储路径与详情|
|`docker volume rm mydata`|删除指定数据卷|
|`docker run -v mydata:/app/data nginx`|挂载数据卷至容器目录|
|`docker run -v /host/path:/container/path nginx`|绑定挂载宿主机本地目录到容器|
|`docker run --mount source=mydata,target=/app/data nginx`|标准 mount 语法挂载数据卷（可读性更高）|
|`docker cp /host/file mynginx:/app/`|宿主机文件复制到容器内|
|`docker cp mynginx:/app/file /host/`|容器内文件复制到宿主机|
---
## 7. 资源清理与磁盘维护

场景：定期清理无用容器、镜像、网络、数据卷释放磁盘

|命令|说明|
|---|---|
|`docker container prune`|删除所有已停止的容器|
|`docker image prune`|清理无标签悬空镜像|
|`docker image prune -a`|删除所有未被容器引用的镜像|
|`docker volume prune`|删除未使用的数据卷（⚠️ 会丢失持久化数据）|
|`docker network prune`|删除未被容器使用的闲置网络|
|`docker system prune`|基础清理：停止容器、悬空镜像、闲置网络|
|`docker system prune -a`|深度全量清理，包含全部未使用镜像（慎用）|
|`docker system df`|统计 Docker 整体磁盘占用明细|
---
## 8. 镜像离线迁移（导入导出）

场景：内网离线环境镜像打包、迁移、恢复

|命令|说明|
|---|---|
|`docker save -o myapp.tar myapp:v1`|将完整镜像（含分层历史）导出 tar 包|
|`docker load -i myapp.tar`|从 tar 文件恢复完整镜像|
|`docker export -o container.tar mynginx`|导出容器单层快照（丢失镜像分层历史）|
|`docker import container.tar mynewimage:v1`|将容器快照导入生成新镜像|
---
## 9. Docker Compose 多容器编排

场景：一键管理多服务应用（前端、后端、数据库等）

|命令|说明|
|---|---|
|`docker-compose up -d`|后台启动 compose 所有服务|
|`docker-compose down`|停止并删除所有服务容器（保留数据卷）|
|`docker-compose ps`|查看 compose 各服务运行状态|
|`docker-compose logs -f`|实时查看全部服务日志|
|`docker-compose logs -f service_name`|仅查看指定服务实时日志|
|`docker-compose restart`|重启全部服务|
|`docker-compose restart service_name`|单独重启某一个服务|
|`docker-compose exec service_name /bin/bash`|进入指定服务容器交互终端|
|`docker-compose build`|重新构建 compose 自定义服务镜像|
|`docker-compose pull`|拉取所有服务最新镜像|
|`docker-compose up -d --scale web=3`|对 web 服务扩容至 3 个运行实例|
---
## 10. 高频组合实用命令

场景：一条命令批量操作容器 / 镜像，提升运维效率

|命令|说明|
|---|---|
|`docker stop $(docker ps -q)`|一次性停止所有正在运行的容器|
|`docker rm $(docker ps -aq)`|删除本地全部容器（含已停止）|
|`docker rmi $(docker images -q)`|删除本地全部镜像|
|`docker stop $(docker ps -q) && docker rm $(docker ps -aq)`|停止并清空所有容器|
|`docker exec -it $(docker ps -q -f name=nginx) /bin/bash`|通过名称过滤直接进入 nginx 容器|
|`docker logs -f --tail 50 $(docker ps -q -f status=running)`|查看所有运行容器最后 50 行实时日志|
